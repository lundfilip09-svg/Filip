Dashboard Setup

Kjør OAuth-flyt én gang for å hente REFRESH_TOKEN for Google Calendar og Zepp. Lim deretter tokens inn som Netlify-miljøvariabler.
Google Calendar OAuth 2.0
Du trenger et Google Cloud-prosjekt med Calendar API aktivert og en OAuth 2.0-klient av typen «Desktop app». Redirect URI: urn:ietf:wg:oauth:2.0:oob (for lokal bruk) eller URL-en til denne siden på Netlify.
1
Gå til console.cloud.google.com → APIs → Calendar API → aktiver. Deretter Credentials → Create credentials → OAuth client ID → Desktop app.
2
Kopier Client ID og Client Secret inn nedenfor, trykk «Åpne Google Auth», logg inn og godkjenn.
3
Google viser en autorisasjonskode. Lim den inn og trykk «Hent tokens».
4
Kopier GOOGLE_REFRESH_TOKEN og legg det inn som miljøvariabel i Netlify.
Legg til i Netlify → Environment Variables

Zepp Health OAuth 2.0
Zepp Open Platform bruker standard OAuth 2.0. Du trenger en registrert app på open-zepp.zepp.com (eller Amazfit developer portal). Redirect URI kan settes til URL-en til denne siden på Netlify, f.eks. https://din-side.netlify.app/setup.html.
1
Registrer app på Zepp Open Platform, noter App ID (client_id) og App Secret (client_secret), og legg til redirect URI.
2
Fyll inn feltene nedenfor og trykk «Åpne Zepp Auth». Logg inn med Zepp-kontoen din.
3
Zepp redirecter tilbake hit med ?code=... i URL-en — koden plukkes opp automatisk.
4
Trykk «Hent tokens» og kopier ZEPP_REFRESH_TOKEN til Netlify.
Legg til i Netlify → Environment Variables
ZEPP_APP_ID–
ZEPP_APP_SECRET•••••••••
ZEPP_REFRESH_TOKEN–
Netlify — legg til miljøvariabler
Gå til app.netlify.com → din site → Site configuration → Environment variables → Add variable. Legg til disse nøklene:
${['GOOGLE_CLIENT_ID','GOOGLE_CLIENT_SECRET','GOOGLE_REFRESH_TOKEN','GOOGLE_CALENDAR_ID (= primary)','ZEPP_APP_ID','ZEPP_APP_SECRET','ZEPP_REFRESH_TOKEN'].map(k => `
${k}
` ).join('')}
